## 0. 插件长什么样

插件最终是塞进面板 `custom_head` 的一段 HTML/JS。两种合法包裹方式:

```html
<!-- 形式 A: 浏览器把它当 <script> 直接跑 (同步逻辑首选) -->
<script>
(() => { /* ... */ })()
</script>
```

```html
<!-- 形式 B: 半行注释 + IIFE,既能被面板识别为脚本,又能在纯 JS 模式下不报错 -->
//<script>
(async () => { /* ... */ })()
//</script >
```

> 注意:形式 B 的关闭标签是 `</script >` (带个空格),这是历史习惯,沿用即可。

## 1. 完整模板

新插件起手就复制这一段,改 `PLUGIN_NAME` 和按钮 `onclick` 里的业务:

```html
//<script>
(async () => {
    const PLUGIN_NAME = 'MyPlugin'
    const LOG = (...a) => console.log(`[${PLUGIN_NAME}]`, ...a)

    // --- runShellWithRoot shim (面板不会自动注入,详见 §6) ---
    if (typeof window.runShellWithRoot !== 'function') {
        window.runShellWithRoot = async (cmd, timeoutMs = 30000) => {
            try {
                const base = (typeof KANO_baseURL !== 'undefined' && KANO_baseURL) ? KANO_baseURL : '/api'
                const headers = { 'Content-Type': 'application/json' }
                if (typeof common_headers !== 'undefined' && common_headers) {
                    Object.assign(headers, common_headers)
                    headers['Content-Type'] = 'application/json'
                }
                const res = await fetch(`${base}/run_shell`, {
                    method: 'POST', headers,
                    body: JSON.stringify({ cmd: String(cmd), timeout: Number(timeoutMs) || 30000 })
                })
                if (!res.ok) return { success: false, content: `HTTP ${res.status}` }
                const d = await res.json()
                return { success: !!d.success, content: d.content || '' }
            } catch (e) { return { success: false, content: String(e && e.message || e) } }
        }
    }

    // --- 等待主页面就绪 (UFI_DATA 是面板挂的全局数据) ---
    const wait = (ms = 100) => new Promise(r => setTimeout(r, ms))
    while (typeof UFI_DATA === 'undefined' || !UFI_DATA.lan_ipaddr) await wait()

    // --- root 自检 (U60Pro 默认 root) ---
    const isRoot = async () => {
        const { content } = await runShellWithRoot('id -u', 3000)
        return (content || '').trim() === '0'
    }

    // --- 注入 UI: 一个标准折叠卡片 ---
    document.querySelector('.functions-container').insertAdjacentHTML('afterend', `
<div id="${PLUGIN_NAME}_BOX" style="width:100%;margin-top:10px">
    <div class="title" style="margin:6px 0"><strong>${PLUGIN_NAME}</strong>
        <div style="display:inline-block" id="${PLUGIN_NAME}_collapse_btn"></div>
    </div>
    <div class="collapse" id="${PLUGIN_NAME}_collapse" data-name="close" style="height:0;overflow:hidden">
        <div class="collapse_box">
            <div id="${PLUGIN_NAME}_actions" style="display:flex;gap:10px;flex-wrap:wrap;padding:8px"></div>
        </div>
    </div>
</div>`)

    // --- 添加按钮 ---
    const actions = document.querySelector(`#${PLUGIN_NAME}_actions`)
    const btn = document.createElement('button')
    btn.classList.add('btn')
    btn.textContent = '点我'
    btn.onclick = async () => {
        if (!(await isRoot())) { createToast('需要 root', 'red'); return }
        createToast('开始执行...')
        const res = await runShellWithRoot('uname -a')
        createToast(res.success ? res.content : '失败:' + res.content,
                    res.success ? 'green' : 'red', 5000)
    }
    actions.appendChild(btn)

    // --- 折叠开关 (三个参数: 触发按钮选择器, 折叠面板选择器, 状态键名) ---
    collapseGen(`#${PLUGIN_NAME}_collapse_btn`,
                `#${PLUGIN_NAME}_collapse`,
                `${PLUGIN_NAME}_collapse_state`)

    LOG('loaded')
})()
//</script >
```

## 2. 面板提供的全局对象 / 函数

> 大多是面板挂在 `window` 上的,直接用即可。**例外:`runShellWithRoot` 面板不注入,必须自带 shim,见 §6。**

### 数据

| 名称 | 说明 |
| --- | --- |
| `UFI_DATA` | 面板全量状态,常用字段 `UFI_DATA.lan_ipaddr` (设备 LAN IP) |
| `KANO_baseURL` | 后端 API 基址,通常就是 `/api` |
| `common_headers` | 已签名的请求头。透传给 fetch 即可,但 POST JSON 时记得把 `Content-Type` 覆盖回 `application/json` |
| `collapseBtn_menu` | 「功能列表」折叠按钮 DOM 引用,在它的 `nextElementSibling.querySelector('.collapse_box')` 里追加 button 是"挂到功能区"的标准入口 |
| `QORS_MESSAGE` | 面板首屏就绪的旗标,部分插件用它判断初始化完成 |

### Toast

```js
createToast('短消息')                       // 默认色,默认时长
createToast('成功了', 'green')              // 颜色: 'green' / 'red' / 'pink'
createToast('注意一下', 'pink', 4000)       // 第三参数 = 持续毫秒
```

```js
// 长任务专用: 一个固定位置、内容可持续刷新的 toast
const { el, close } = createFixedToast('my_toast_id',
    `<pre style="white-space:pre-wrap;min-width:300px;text-align:center;">等待中...</pre>`,
    '' /* 颜色, 可留空 */)
// el 是该 toast 的 DOM,可以直接 el.innerHTML = ...
// close() 手动关闭;同一 id 再调一次会复用,不会叠加
```

> `createFixedToast` 不止用于"刷日志",还能塞按钮做**富交互对话框**(确认框 / 风险警告 / 结果详情)。完整三种模式见 §7.6。

### Modal

```js
showModal('#myModalId')   // 显示
closeModal('#myModalId')  // 关闭 (全局可调,不挂 . prefix 也行)
```

Modal 的 DOM 结构(直接 `appendChild` 到 `document.body`):

```html
<!-- 一个带遮罩的标准模态 -->
<div class="mask" id="myModal" style="display:none;top:0;left:0;right:0;bottom:0;">
    <div class="modal" style="width:90%;max-width:800px;">
        <div class="title">标题</div>
        <p>正文</p>
        <div class="btn" style="text-align:right;">
            <button onclick="closeModal('#myModal')">关闭</button>
        </div>
    </div>
</div>
```

> `mask` 一定写 `top/left/right/bottom:0`,否则 `appendChild(document.body)` 后 fixed 元素会落到页底。

### 折叠面板

```js
// 参数: 触发按钮选择器, 折叠主体选择器, 状态键名(用作 localStorage key),
//        可选 onChange 回调
collapseGen('#xxx_btn', '#xxx_collapse', 'xxx_state', (e) => { /* 可选 */ })
```

### Shell 执行

```js
// cmd: 字符串, 走 BusyBox sh
// timeoutMs: 超时, 默认 30000
const { success, content } = await runShellWithRoot('uname -a', 5000)
```

后端协议(`/api/run_shell`):请求 `{cmd:string, timeout:number(ms)}`,响应 `{success:bool, content:string}`。

> ⚠️ **`runShellWithRoot` 面板不会自动注入**,插件**必须**在使用前自带 shim,否则会报 `runShellWithRoot is not defined`。完整 shim 见 §6,复制粘贴即可。

| 名称 | 说明 |

## 4. 系统侧约定 (OpenWrt + BusyBox)

### 4.1 跨发行版 shell 指令注意

* **判 root**:`id -u` == "0",**不要用** `whoami` (BusyBox 可能裁掉)。
* **杀进程**:`pidof <name>` 拿 PID,逐个 `kill`,过 0.5s 再 `pidof` 复查,残留的用 `kill -9`。**不要** `ps -ef | grep`,BusyBox 的 ps 字段顺序与 Linux 不同。
* **超时**:用 `runShellWithRoot` 的 `timeoutMs` 参数,**不要**调系统 `timeout` 命令。
* **架构判断**:`uname -m` → 自己映射:`x86_64→amd64`,`aarch64→arm64`,`loongarch64→loong64`,`mips64le→mips64le`,`s390x→s390x`。
* **整段 shell 优先 POSIX 子集**,不依赖 bash 扩展。

### 4.2 持久化路径

| 用途 | 路径 |
| --- | --- |
| **插件数据根目录 (统一前缀)** | **`/data/plugins/`** |
| **子插件自有目录** | **`/data/plugins/<plugin_name>/`** |
| 上传 / 可下载产物 | `/data/plugins/uploads/`,前端 `<a href="/api/uploads/xxx">` |
| 配置 | `/data/plugins/<plugin_name>/config.json` |
| 数据库 | `/data/plugins/<plugin_name>/data.db` |
| 运行日志 | `/tmp/xxx.log` 即可,重启清掉无所谓 |

> 写入前 `mkdir -p` 兜底,目录不存在不会自动建。
>
> 注意:`/data/ufi-tools/` 是**面板自身**的数据/配置目录,**不要把插件数据塞进去**——避免和面板升级冲突,也避免一卸载就把面板配置带走。统一往 `/data/plugins/` 写。

### 4.3 开机自启 —— 双路径探测:`/sdcard/...` 优先,`/data/ufi-tools/sdcard/...` 兜底

> U60Pro 的 `service.sh` 开机时会 bind `/data/ufi-tools/sdcard` 到 `/sdcard`(或 ext4 loop 兜底),然后逐行执行 `/sdcard/ufi_tools_boot.sh`。
>
> 大多数运行时刻 `/sdcard` 已挂好可写,但有些极早期场景(bind 还没建/挂失败/分区只读)写 `/sdcard` 会失败。这时写它的底层 bind 源 `/data/ufi-tools/sdcard/ufi_tools_boot.sh` 同样生效,**两个路径透过 bind 是同一份文件**。
>
> **不要碰 `/etc/rc.local`** — 它是固件层文件,改坏了进不去系统;某些底层驱动确实需要早于 service.sh 触发的极端场景才考虑,且要明确知道在做什么。

#### 探测出当前可写路径 (JS 侧)

```js
let BOOT_SH = '/data/ufi-tools/sdcard/ufi_tools_boot.sh'   // 兜底默认
const r = await runShellWithRoot(`
if [ -d /sdcard ] && touch /sdcard/.uf_test 2>/dev/null; then
    rm -f /sdcard/.uf_test; echo /sdcard/ufi_tools_boot.sh
else
    mkdir -p /data/ufi-tools/sdcard 2>/dev/null
    echo /data/ufi-tools/sdcard/ufi_tools_boot.sh
fi`, 3000)
const p = (r.content || '').trim()
if (p) BOOT_SH = p
```

#### 幂等的安装/卸载模板

```sh
# 安装: 父目录兜底 + 不存在则建骨架 + grep -qF 去重追加
mkdir -p $(dirname ${BOOT_SH}) 2>/dev/null
if [ ! -f ${BOOT_SH} ]; then
    printf '#!/bin/sh\n# UFI-TOOLS-U60Pro plugin boot\n' > ${BOOT_SH}
    chmod 755 ${BOOT_SH}
fi
grep -qF '<我的启动行>' ${BOOT_SH} \
    || echo '<我的启动行>' >> ${BOOT_SH}

# 卸载: sed -i 删除带本插件关键词的所有行
[ -f ${BOOT_SH} ] && sed -i '/<我的关键词>/d' ${BOOT_SH}
```

> 关键词起得越独特越好(用插件可执行文件名通常就够),避免误伤别人的行。
>
> 实战参考:`WiFi节能控制插件.js` / `屏幕壁纸编辑器.js` / `OpenList插件.js` 顶部的 `BOOT_SH 探测` 代码块,直接复制即可。

### 4.4 fetch 与鉴权

所有 `fetch` 都会经面板的 auth 中间件:

* 未登录返回 **401**,前端会自动跳转,不需要插件管。
* `common_headers` 已经把签名头塞好,POST JSON 时把 `Content-Type` 覆盖回 `application/json`(`common_headers` 里可能带 referer/host 之类的)。

### 4.5 编辑大段文本文件 —— 用 base64

shell 里写多行 / 含 `$` 反引号 单双引号的内容,转义会爆炸。统一约定:

```js
// JS 侧 UTF-8 安全编码
const toB64 = (s) => {
    const bytes = new TextEncoder().encode(s)
    let bin = ''; for (let i = 0; i < bytes.length; i++) bin += String.fromCharCode(bytes[i])
    return btoa(bin)
}

// shell 侧解码写回, 写前先备份
await runShellWithRoot(
    `cp ${TARGET} ${TARGET}.bak 2>/dev/null; ` +
    `echo '${toB64(content)}' | base64 -d > ${TARGET} && chmod 755 ${TARGET} && echo ok`
)
```

## 5. 三个"必装"特殊插件

它们改变其它插件的运行环境,加载顺序敏感,文件名里都写明了位置要求。

### 5.1 `【请将插件置顶】插件收纳箱.txt`

劫持 `HTMLElement.prototype.appendChild`:把所有原本要塞进「功能列表」折叠区的 button,改塞进一个统一的「插件收纳箱」模态。还 hook 了 `showModal`/`closeModal` —— 用户打开别的插件 modal 时,收纳箱会自动关掉。

→ 写普通插件时**默认按钮挂载点不变**,直接 `appendChild` 到 `collapseBtn_menu.nextElementSibling.querySelector('.collapse_box')`,收纳箱会接管。

### 5.2 `点击"功能列表"标题切换按钮可见性1.1(请放在插件最后).js`

扫描 `.functions-container` 下所有 `button` / `select`,克隆一份做开关,状态存在 `localStorage.buttonsState`。点功能列表的标题(`#collapseBtn_menu > strong`)弹出开关面板。

→ **必须最后加载**,否则后来的按钮不会出现在开关面板里。

### 5.3 `首页插件移动到状态区后(请将插件顺序置顶).txt`

劫持 `insertAdjacentHTML`:任何对 `.functions-container` 的 `insertAdjacentHTML` 都自动改写为对 `.status-container`。

→ 想用就**置顶**,所有走 §1 模板的插件会被一起搬到状态区。

## 6. `runShellWithRoot` shim (**必须**)

面板**不会**注入 `runShellWithRoot`,只要插件里用到 shell,起手第一步就**先**把这段 shim 贴上。`OpenList插件.js` / `AT指令插件.js` / `rc_local编辑器插件.js` 都带这段,删掉就跑不起来。

```js
if (typeof window.runShellWithRoot !== 'function') {
    window.runShellWithRoot = async (cmd, timeoutMs = 30000) => {
        try {
            const base = (typeof KANO_baseURL !== 'undefined' && KANO_baseURL) ? KANO_baseURL : '/api'
            const headers = { 'Content-Type': 'application/json' }
            if (typeof common_headers !== 'undefined' && common_headers) {
                Object.assign(headers, common_headers)
                headers['Content-Type'] = 'application/json'  // common_headers 可能塞了别的 CT
            }
            const res = await fetch(`${base}/run_shell`, {
                method: 'POST', headers,
                body: JSON.stringify({ cmd: String(cmd), timeout: Number(timeoutMs) || 30000 })
            })
            if (!res.ok) return { success: false, content: `HTTP ${res.status}` }
            const data = await res.json()
            return { success: !!data.success, content: data.content || '' }
        } catch (e) {
            return { success: false, content: String(e && e.message || e) }
        }
    }
}
```

> `typeof window.runShellWithRoot !== 'function'` 这个守卫保留:万一未来面板真注入了原生实现,插件不会覆盖它。

```js
const { el, close } = createFixedToast('mytask', `<pre>等待日志中...</pre>`)
const t0 = performance.now()
const timeout = 100 * 1000
const timer = setInterval(async () => {
    const r = await runShellWithRoot(`cat /tmp/mytask.log 2>/dev/null`)

    el.style.overflow = 'auto'

        clearInterval(timer); close()
    }
}, 1000)
```

### 7.5 AT 指令 (对齐 `at_cmd.rs::PORTS` 与 `send` 模板)

```js
const PORTS = ['/dev/at_mdm0','/dev/at_mdm1','/dev/at_usb0','/dev/smd7','/dev/smd11']
const shq = s => String(s).replace(/'/g, "'\\''")

const runOnPort = (port, cmd, waitSecs) => runShellWithRoot(
    `cat ${port} & PID=$! ; sleep 0.3 ; ` +
    `echo -e '${shq(cmd)}\\r' > ${port} ; ` +
    `sleep ${waitSecs} ; kill $PID 2>/dev/null`,
    (waitSecs + 5) * 1000
)

const detectPort = async () => {
    for (const p of PORTS) {
        const exists = await runShellWithRoot(`[ -e ${p} ] && echo ok`, 2000)
        if (((exists.content||'').trim()) !== 'ok') continue
        const r = await runOnPort(p, 'AT', 1)
        if ((r.content || '').includes('OK')) return p
    }
    return null
}
// 检测到的端口建议缓存到 localStorage,使用前 [ -e ] 校验一次仍然存在
```

### 7.6 富交互对话框 / 风险提示 (createFixedToast 进阶)

`createToast` 只能弹短消息,做不了"用户必须点确认"的对话。要做删除确认、风险警告、结果详情这类需要交互的弹窗,用 `createFixedToast` + HTML 按钮 + `Promise` 包装。

#### a. 二选一确认 (确认/取消)

```js
const promptConfirm = (title, body) => new Promise((resolve) => {
    const { el, close } = createFixedToast('plugin_confirm_toast', `
<div style="pointer-events:all;width:88vw;max-width:420px;">
```

#### b. 倒计时强制阅读 (高风险操作专用)

按钮先 `disabled`,N 秒后才能点,强制用户至少看一眼说明。

```js
const promptHighRisk = (message, waitSecs = 3) => new Promise((resolve) => {
    let cd = waitSecs
    const { el, close } = createFixedToast('plugin_risk_toast', `
<div style="pointer-events:all;width:90vw;max-width:480px;">
    <div class="title" style="margin:0;color:#ffb3b3;">高风险操作提示</div>
    <div style="margin-top:12px;padding:12px;border-radius:12px;
                background:rgba(255,70,70,.12);border:1px solid rgba(255,90,90,.28);
                font-size:.72rem;line-height:1.8;">
        ${message}
    </div>
    <div style="margin-top:12px;text-align:right;">
        <button class="risk-go" disabled>请等待 ${cd}s</button>
        <button class="risk-cancel">取消</button>
    </div>
</div>`)
    const $go = el.querySelector('.risk-go')
    const timer = setInterval(() => {
        cd -= 1
        if (cd > 0) { $go.textContent = `请等待 ${cd}s`; return }
        clearInterval(timer); $go.disabled = false
        $go.textContent = '我已知晓风险,继续'
    }, 1000)
    const done = (v) => { clearInterval(timer); close(); resolve(v) }
    $go.onclick = () => done(true)
    el.querySelector('.risk-cancel').onclick = () => done(false)
})

// 用法
if (!(await promptHighRisk('刷写分区可能导致设备无法启动...'))) return
await flashPartition()
```

#### c. 结果详情展示 (长内容 + 关闭回调)

dd / curl / 任何长输出命令的结果都适合用这个展开;关闭时可链式触发下一步 (如"删除缓存"提示)。

```js
const showResult = (title, content, onClose) => {
    const { el, close } = createFixedToast('plugin_result_toast', `
<div style="pointer-events:all;width:90vw;max-width:520px;">
    <div class="title" style="margin:0">${title}</div>
    <pre style="margin-top:10px;max-height:200px;overflow:auto;
                white-space:pre-wrap;word-break:break-all;
                background:rgba(0,0,0,.22);padding:10px;border-radius:10px;
                font-size:.66rem;">${content}</pre>
    <div style="margin-top:12px;text-align:right;">
        <button class="close">关闭</button>
    </div>
</div>`)
    el.querySelector('.close').onclick = () => { onClose?.(); close() }
}
```

#### 必守的几条约定

| 项 | 说明 |
|---|---|
| `pointer-events:all` 必须显式开 | 外层 `createFixedToast` 容器默认 `pointer-events:none` (避免遮挡其它点击),内容根需要打开,否则按钮死活点不到 |
| ID 起得独特 | 同一 id 再调一次会**复用 DOM** 而不是叠加;所以同时只能有一个相同 id 的 toast。跨插件用各自前缀避免撞车 (如 `pm_xxx`、`devui_xxx`) |
| `Promise` 包装异步 | 配 `await` 链式调用最清爽;`resolve` 前一定 `close()` + `clearInterval` 清掉定时器,否则定时器会泄漏 |
| 倒计时按钮先 `disabled` | 强制用户至少看 N 秒,防"惯性手快"误触。文案要带秒数 (`请等待 3s`) 让用户知道在等什么 |
| 危险按钮配色 | 标题 `color:#ffb3b3`、容器红底 `rgba(255,70,70,.12)` + 红边 `rgba(255,90,90,.28)`;跟普通信息态视觉拉开距离 |
| `el.querySelector` 在 el 内查 | `el` 已是该 toast 的根,内部 class 只需在 toast 内唯一即可,不必加全局前缀 |
| 必须 `close()` 才会消失 | 跟 `createToast` 自动消失不同,`createFixedToast` 不接 timeout 参数,**手动关闭**才隐藏;漏掉用户会卡死 |

#### 几种模式什么时候选

| 用 | 选哪个 |
|---|---|
| 一次性短反馈 (成功/失败/告知)| `createToast(msg, color, ms)` |
| 长任务进度刷新 (滚动日志) | `createFixedToast` + 定时器更新 `el.innerHTML` (§7.2) |
| 单击就走的危险按钮防误触 | "连点 N 次"(§7.3),不弹对话框 |
| 需要用户点确认/取消 | 富对话框 a 或 b (§7.6) |
| 高风险操作 (刷分区/卸载/格式化) | 富对话框 b (倒计时强制阅读) |
| 命令长输出展示 + 后续动作 | 富对话框 c (§7.6) |
